# 项目最精简架构与启动说明

## 1. 最精简项目架构（按职责）

```text
RemotePlatform/
├─ docker-compose.yml        # 统一启动入口（全栈一条命令）
├─ backend/                  # FastAPI 后端：健康检查 + WebSocket 遥测/控制
│  ├─ Dockerfile             # 后端镜像构建
│  ├─ app/
│  │  └─ main.py             # 核心后端入口（/health, /ws）
│  └─ requirements.txt       # 后端依赖
├─ frontend/                 # React 前端：控制面板 + 3D机械臂 + 指标展示
│  ├─ Dockerfile             # 前端镜像构建（Node 构建 + Nginx 运行）
│  ├─ nginx/
│  │  └─ default.conf        # 前端 Nginx 配置（含 SPA 路由回退）
│  ├─ src/
│  │  ├─ main.tsx            # 前端入口（挂载 React + MUI Theme）
│  │  ├─ App.tsx             # 主页面与核心业务逻辑（按钮、WebSocket、3D渲染）
│  │  └─ theme.ts            # MUI 主题配置
│  └─ package.json           # 前端依赖与脚本（dev/build/preview）
```

## 2. 重点代码文件与作用

### `backend/app/main.py`
- 提供 `GET /health` 健康检查接口。
- 提供 `WebSocket /ws` 实时通道。
- `receiver()`：接收前端控制指令（如 `robot_joint_control`、`robot_reset`），更新关节状态。
- `sender()`：周期发送 telemetry（时延、抖动、丢包、带宽、关节角）。

### `frontend/src/main.tsx`
- 前端启动入口。
- 挂载 `App` 到 `#root`。
- 注入 `ThemeProvider` 和 `CssBaseline` 统一 UI 基线。

### `frontend/src/App.tsx`
- 页面主逻辑集中在该文件：
- 控制区：关节步进、前进后退、抓取松开、复位。
- 通信区：建立 WebSocket，发送控制命令，接收 telemetry。
- 3D区：React Three Fiber + drei 渲染机械臂（相机、光照、阴影、环境贴图、OrbitControls）。
- 指标区：展示时延、抖动、丢包、带宽，并按阈值上色。

### `frontend/package.json`
- `npm run dev`：开发模式。
- `npm run build`：生产构建（`tsc -b && vite build`）。

### `README.md`
- 项目快速启动说明与访问地址。

## 3. 最精简启动方式（唯一）

在项目根目录执行：

先确认 Docker Desktop 已启动（Docker Engine 为 Running）。

```bash
docker compose up --build
```

启动后访问：
- 前端：`http://localhost:8080`
- 后端健康检查：`http://localhost:8000/health`

停止项目：

```bash
docker compose down
```

## 4. 运行链路（核心）

- 前端按钮 -> WebSocket 下发控制命令到后端 -> 后端更新关节状态并持续回传 telemetry -> 前端实时刷新 3D机械臂与仪表数据。
